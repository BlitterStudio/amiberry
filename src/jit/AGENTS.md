# src/jit/ — JIT Compiler

## OVERVIEW

JIT compiler translating M68K code to native ARM64 or x86-64 at runtime. Most complex subsystem. Two backends sharing common dispatcher. Use `amiberry-arm64-jit` for ARM64 issues and `amiberry-x86-jit` for x86-64 allocator, high-natmem, or RIP-relative issues.

## JIT PIPELINE

```
M68K Code Block
  → [Liveflags Analysis] backward pass: which flags needed?
  → [Instruction Compilation] comptbl[opcode]() per instruction
  → [Register Allocation] virtual → native register mapping
  → [Flag Optimization] dont_care_flags() skips unneeded flag computation
  → [Block Chaining] dependency links to successor blocks
  → [Finalization] flush registers, emit cycle counter
  → Native ARM64/x86 Code
```

## CONVENTIONS

- **PC_P (register 16)** is the ONLY 64-bit virtual register — all others hold 32-bit M68K values
- **19 fix points** in ARM64 backend ensure 64-bit pointer correctness
- **MIDFUNC layer** = platform-independent intermediate; codegen layer = platform-specific
- **Block optimization levels**: execute `optcount[level]` times before compilation (default: 4)
- **Checksum invalidation**: blocks checksummed; memory writes trigger selective recompilation

## ANTI-PATTERNS

- **NEVER** truncate 64-bit pointers to 32-bit — use `uintptr` for address calculations
- **NEVER** modify `compemu_arm.cpp` / `compemu_x86.cpp` directly — they're generated by `gencomp`
- **Inter-block flag optimization is DISABLED** (`#if 0` in `compile_block()`) on ARM64 — do not re-enable without the two-pass liveflags fix
- **ARM carry is inverted** vs M68K — tracked by `flags_carry_inverted`
- **x86-64**: high natmem is allowed; `R_MEMSTART` holds `natmem_offset`, `PC_P` stays pointer-width, and non-`PC_P` virtual registers stay 32-bit

## NOTES

- Use `amiberry-arm64-jit` for ARM64 JIT debugging and `amiberry-x86-jit` for x86-64 JIT debugging
- Android: Code memory allocated RWX permanently (no W^X toggling)
- x86-64: Anchor-based allocator places JIT code/pools within RIP-relative range of the active JIT/data anchor
- Page 0 DMA guard (ARM64): SIGSEGV/SIGTRAP pair protects exception vectors during Kickstart init
