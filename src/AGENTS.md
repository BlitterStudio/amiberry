# src/ — Emulation Core

## OVERVIEW

~1100 C/C++ files implementing the Amiga emulation engine. WinUAE-derived core with Amiberry platform customizations.

## STRUCTURE

## WHERE TO LOOK

| Task | Location | Notes |
|------|----------|-------|
| Custom chip (DMA, copper, blitter) | `custom.cpp` | 12.7K lines, cycle-accurate |
| CPU interpreter | `newcpu.cpp` + `cpuemu_*.cpp` | cpuemu files are generated |
| Memory banks | `memory.cpp` + `include/memory.h` | `addrbank` struct, 65536-bank model |
| Config parsing | `cfgfile.cpp` | 10.5K lines, add options in `include/options.h` |
| Floppy emulation | `disk.cpp` | Index hack, AMAX speedup hack |
| Audio | `audio.cpp` | DMA wait hack, low period hack |
| Filesystem | `filesys.cpp` | Known crash on concurrent host FS modification |
| Archive handling | `zfile_archive.cpp` → `archivers/` | Dispatcher pattern per format |
| Network | `slirp_uae.cpp` → `slirp/` | DHCP/TCP/UDP/DNS |
| Expansion boards | `expansion.cpp` | Z3 mapping hacks, KS 1.2 boot hack |

## CONVENTIONS

- Core files track **WinUAE upstream** — use `winuae-amiberry-merge` skill for porting
- Amiberry-specific code guarded by `#ifdef AMIBERRY`
- Generated files (`cpuemu_*`, `linetoscr_*`, `cpustbl`, `blitfunc`, `blittable`) — regenerate via `gencpu.cpp`/`genblitter.cpp`
- Two-state prefs: `currprefs` (active) vs `changed_prefs` (pending) — atomic config changes
- Event-driven timing: `eventtab[ev_max]` with priorities `ev_sync > ev_cia > ev_misc > ev_audio`

## ANTI-PATTERNS

- **DO NOT** edit `cpuemu_*.cpp` — they're generated by `gencpu.cpp`
- **DO NOT** enable `#if 0` blocks without understanding context
- Audio hacks (`usehacks()`, `TOD_HACK`, `cia_timer_hack`) are **essential** for compatibility
- `filesys.cpp` has 5 documented limitations (line 15-23) — doesn't check for sane values
- FPU SNAN detection **broken** in `fpp_native.cpp` — cannot be fixed via host FPU
